## Calculadora Científica

### Problema:
A proposta era criar uma calculadora científica funcional que permitisse ao usuário realizar múltiplas operações matemáticas em sequência, com menu de navegação e suporte a operações como fatorial, funções de segundo grau, conversão de bases numéricas, concatenação de strings e cálculo de juros. O desafio incluía a documentação completa do sistema com manual de usuário e instruções de uso.

### Solução Entregue pela Equipe:
A equipe desenvolveu a calculadora em duas linguagens: inicialmente em VisualG/Portugol para a lógica algorítmica e, posteriormente, em TypeScript para uma implementação mais robusta. O sistema contempla todas as operações solicitadas com um menu interativo, garantindo que o usuário possa navegar entre as funções sem reiniciar o programa. A documentação foi produzida ao longo das três sprints.

[Repositório do Projeto](https://github.com/BugBusters-Suricatos/1st-Semester)

> - **Git e GitHub**: Essenciais para controle de versão e colaboração entre os membros da equipe.

### Em 2025-1

## Gerenciador de Horários de Aula

### Empresa Parceira: [Projeto Interno — FATEC SJC]()

### Problema:
A organização de horários de aula é frequentemente complexa ao lidar com professores compartilhados entre cursos e semestres. O projeto visava criar uma aplicação que gerenciasse horários de forma otimizada, evitando conflitos entre professores e disciplinas, com sugestões inteligentes de alocação e gerenciamento completo de cursos, semestres e professores.

### Solução Entregue pela Equipe:
A equipe desenvolveu um sistema desktop com interface gráfica em JavaFX, backend em Java e banco de dados MySQL. A aplicação permite que coordenadores cadastrem cursos, disciplinas e professores, definam horários por semestre e recebam alertas automáticos em caso de sobreposição. A Sprint 3 adicionou sugestão de alocação inteligente de horários. O sistema foi acompanhado de manual do usuário, guia de instalação e modelo ER documentado.

[Repositório do Projeto](https://github.com/BugBusters-Suricatos/2nd-Semester)

#### Tecnologias Utilizadas

> - **Java**: Linguagem principal para o desenvolvimento do backend e lógica de negócio do sistema.
> - **JavaFX**: Framework utilizado para construção da interface gráfica desktop, com telas de cadastro, visualização de horários e alertas de conflito.
> - **MySQL**: Banco de dados relacional utilizado para armazenar cursos, disciplinas, professores e horários.
> - **JavaScript**: Utilizado na Sprint 3 para implementação de funcionalidades adicionais e ajustes visuais na interface.
> - **Git e GitHub**: Controle de versão e colaboração entre os membros da equipe.

### Em 2025-2

## Plataforma de Mapeamento de Competências — Altave

### Empresa Parceira: [Altave](https://front-altave.vercel.app/)

### Problema:
A Altave enfrentava dificuldade em identificar habilidades específicas entre seus colaboradores à medida que a empresa crescia. A ausência de uma plataforma centralizada tornava impossível para gestores visualizarem as competências da equipe de forma estratégica, dificultando a formação de times e a tomada de decisões sobre desenvolvimento de talentos.

### Solução Entregue pela Equipe:
A equipe desenvolveu uma plataforma web no modelo de "LinkedIn interno", onde colaboradores cadastram seus perfis profissionais com competências, certificações e experiências, e gestores podem buscar habilidades específicas, adicionar avaliações e visualizar relatórios de competências por departamento. O sistema possui três níveis hierárquicos (Diretor, Supervisor, Colaborador) e foi entregue com deploy em produção via Railway (backend) e Vercel (frontend).

[Repositório do Projeto](https://github.com/LegacyTec/Documentacao)

#### Tecnologias Utilizadas

> - **Java 17 + Spring Boot**: Utilizados no desenvolvimento da API REST do backend, com JPA/Hibernate para mapeamento objeto-relacional.
> - **React + TypeScript**: Utilizados no desenvolvimento do frontend, proporcionando uma interface dinâmica e tipagem segura.
> - **MySQL**: Banco de dados relacional normalizado em 3FN para armazenamento de perfis, competências e estrutura hierárquica.
> - **Vite**: Build tool utilizada no frontend para desenvolvimento ágil e otimização de produção.
> - **Tailwind CSS v4**: Utilizado para estilização da interface com classes utilitárias.
> - **Git e GitHub**: Controle de versão com GitFlow simplificado e padrão de Conventional Commits.
